1 Motivations


Note 1. We introduce concepts of convexity and smoothness that facilitate the (theoretical) analysis of the learning problems and their solution that we will discuss (eg stochastic gradient descent) later on. Also learning problems with such characteristics can be learned more efficiently.

Note 2. Most of the ML problems discussed in the course (eg, Artificial neural networks, Gaussian process regression) are usually non-convex.

Note 3. To overcome this problem, we will introduce the concept of surrogate loss function that allows a non-convex problem to be handled with the tools introduced int he convex setting.

2 Convex learning problems

Definition 4. Convex learning problem is a learning problem ( H , Z , ℓ ) that the hypothesis class H is a convex set, and the loss function ℓ is a convex function for each example z ∈ Z .

Example 5. Multiple linear regression <w,x> → y with y ∈ R , hypothesis class H ={ w ∈ R d } and loss ℓ ( w,( x,y ) ) = ( <w,x>-y ) 2 with w * = arg min ∀ w E ( <w,x>-y ) 2 or w ** = arg min ∀ w 1 m ∑ i=1 m ( <w, x i >-y ) 2 is a convex learning problem for reasons that will be discussed below.

Definition 6. A set C is convex if for any u,v ∈ C , the line segment between u and v is contained in C . Namely,
  • for any u,v ∈ C and for any α ∈ [ 0,1 ] we have that α u+( 1− α ) v ∈ C .

6 Non-convex learning problems (surrogate treatment)

Remark 27. A a learning problem may involve non-convex loss function ℓ ( w,z ) which implies a non-convex risk function R g ( w ) . However, our learning algorithm will be analyzed in the convex setting. A suitable treatment to overcome this difficulty would be to upper bound the non-convex loss function ℓ ( w,z ) by a convex surrogate loss function ℓ ˜ ( w,z ) for all w , and use ℓ ˜ ( w,z ) instead of ℓ ( w,z ) .
Example 28. Consider the binary classification problem with inputs x ∈ X , outputs y ∈ { -1,+1 } ; we need to learn w ∈ H from hypothesis class H ⊂ R d with respect to the loss ℓ ( w,( x,y ) ) = 1 ( y ⟨ w,x ⟩ ≤ 0 ) with y ∈ R , and x ∈ R d . Here ℓ ( ⋅ ) is non-convex. A convex surrogate loss function can be ℓ ˜ ( w,( x,y ) ) = max ( 0,1-y ⟨ w,x ⟩ ) which is convex (Example 14) wrt w . Note that:
  • ℓ ˜ ( w,( x,y ) ) is convex wrt w ; because max ( ⋅ ) is convex
  • ℓ ( w,( x,y ) ) ≤ ℓ ˜ ( w,( x,y ) ) for all w ∈ H
Then we can compute w ˜ * = arg min ∀ x ( R ˜ g ( w ) ) = arg min ∀ x ( E ( x,y ) ∼ g ( max ( 0,1-y ⟨ w,x ⟩ ) ) ) instead of w * = arg min ∀ x ( R g ( w ) ) = arg min ∀ x ( E ( x,y ) ∼ g ( 1 ( y ⟨ w,x ⟩ ≤ 0 ) ) ) Of course by using the surrogate loss instead of the actuall one, we introduce some approximation error in the produced output w ˜ * ≠ w * .

Remark 29. (Intuitions...) Using a convex surrogate loss function instead the convex one, fascilitates computations but introduces extra error to the solution. If R g ( ⋅ ) is the risk under the non-convex loss, R ˜ g ( ⋅ ) is the risk under the convex surrogate loss, and w ˜ alg is the output of the learning algorithm under R ˜ g ( ⋅ ) then we have the upper bound R g ( w ˜ alg ) ≤ min w ∈ H ( R g ( w ) ) ⏟ I + ( min w ∈ H ( R ˜ g ( w ) ) - min w ∈ H ( R g ( w ) ) ) ⏟ II + ε ⏟ III where term I is the approximation error measuring how well the hypothesis class performs on the generating model, term II is the optimization error due to the use of surrogate loss instead of the actual non-convex one, and term III is the estimation error due to the use of a training set and not the whole generation model.
