Remark 7. After T iterations of the GD algorithm, the output of GD can be (but not a exclusively),
  1. the average (after discarding the first few iterations of w ( t ) for stability reasons) w GD ( T ) = 1 T ∑ t=1 T w ( t ) (2.5)
  2. or the best value discovered w GD ( T ) = arg min ∀ w t ( f( w ( t ) ) )
  3. or the last value discovered w GD ( T ) = w ( T )

Note 8. GD output converges to a local minimum, w GD ( T ) → w * (in some sense), under different sets of regularity conditions (some are weaker other stronger). Section 4 has a brief analysis.

Remark 9. The parameter η t is called learning rate, step size, or gain. { η t } is a non-negative sequence and it is chosen by the practitioner. In principle, regularity conditions (Note 8) often imply restrictions on the decay of { η t } which guide the practitioner to parametrize it properly. Some popular choices of learning rate η t are:
  1. constant; η t = η , for where η >0 is a small value. The rationale is that GD chain { w t } performs constant small steps towards the (local) minimum w * and then oscillate around it.
  2. decreasing and converging to zero; η t ssearrow with lim t → ∞ η t =0 E.g. η t = ( C t ) ς where ς ∈ (0.5,1] and C>0 . The rationale is that GD algorithm starts by performing larger step at the begining to explore the area for discovering possible minima. Also it reduces the size of those steps with the iterations such that eventually when the chain { w t } is close to a possible minimum w * value to converge and do not overshoot.
  3. decreasing and converging to a tiny value τ * ; η t ssearrow with lim t → ∞ η t = τ * E.g. η t = ( C t ) ς + τ * with ς ∈ (0.5,1] , C>0 , and τ * ≈ 0 . Same as previously, but the algorithm aims at oscillating around the detected local minimum.
  4. constant until an iteration T 0 and then decreasing; eg η t = ( C max ( t, T 0 ) ) ς with ς ∈ (0.5,1] and C>0 , and T 0 <T . The rationale is that at the first stage of the iterations (when t ≤ T 0 ) the algorithm may need a constant large steps for a significant number of iterations T 0 in order to explore the domain; and hence in order for the chain { w t } to reach the area around the (local) minimum w * . In the second stage, hoping that the chain { w t } may be in close proximity to the (local) minimum w * the algorithm progressively performs smaller steps to converge towards the minimum w * . The first stage ( t ≤ T 0 ) is called burn-in; the values { w t } produced during the burn-in ( t ≤ T 0 ) are are often discarded/ignored from the output of the GD algorithm.
  • Parameters C, ς , τ * , T 0 may be chosen based on pilot runs.  

Remark 10. There are several practical termination criteria that can be used in step 2. in GD Algorithm 1. They are to terminate the recursion in practice. Some popular termination criteria are
  1. terminate when the gradient is sufficiently close to zero; i.e. if ∥ ∇ f( w ( t ) ) ∥ ≤ ε for some pre-specified tiny ε >0 then STOP
  2. terminate when the chain w ( t ) does not change; i.e. if ∥ w ( t+1 ) - w ( t ) ∥ ≤ ε ∥ w ( t ) ∥ for some pre-specified tiny ε >0 then STOP
  3. terminate when a pre-specified number of iterations T is performed; i.e. if t ≥ T then STOP
obviously the first two are preferable to the latter.

3 GD for non-differentiable functions (using sub-gradients)

In several learning problems the function to be minimized is not differentiable. GD can be extended to address such problems with the use of subgradients.
Definition 11. Vector v is called subgradient of a function f:S → R at w ∈ S if ∀ u ∈ S,f( u ) ≥ f( w ) + ⟨ u-w,v ⟩ (3.1)

Notation 12. The set of subgradients of function f:S → R at w ∈ S is denoted by ∂ f( w ) .

Note 13. Essentially there may be more than one subgradients of the function at a specific point. As seen by (3.1), subgradients are the slopes of all the lines passing through the point ( w,f( w ) ) and been under the function f( ⋅ ) .
